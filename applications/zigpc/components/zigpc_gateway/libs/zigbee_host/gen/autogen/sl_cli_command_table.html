ota-server

ota-server related commands.
ota-server notify u16 u8 u8 u8 u16 u16 u32 otaImageNotifyCommand
Sends an OTA Image Notify message to the specified destination indicating a new version of an image is available for download.
Arguments
  • u16The node ID (can be a broadcast address) to which this OTA Notify message should be sent
  • u8Target endpoint for the OTA Notify message (only meaningful for non-broadcast target destination)
  • u8Specifies which parameters to include in the OTA Notify cluster command payload (0 = jitter value only; 1 = jitter and manufacturer id; 2 = jitter, mfr id, and device id; 3 = jitter, mfr id, device id, and firmware version)
  • u8Corresponds to QueryJitter parameter in the OTA Upgrade cluster specification. Indicates whether the client receiving Image Notify Command should send in Query Next Image Request command or not.
  • u16Manufacturer ID for the image being advertised (should match the mfr ID in the OTA files header)
  • u16Image type ID for the image being advertised (should match the image type ID from the OTA files header)
  • u32Firmware version of the image being advertised (should match the version from the OTA files header)
ota-server upgrade u16 u8 u16 u16 u32 otaSendUpgradeCommand
Instructs a device to upgrade now.
Arguments
  • u16Short destination to send message
  • u8Endpoint destination to send message
  • u16Manufacturer ID for the image (0xFFFF for wildcard)
  • u16Image type for the image (0xFFFF for wildcard)
  • u32File version for the image (0xFFFFFFFF for wildcard)

ota-server policy

policy related commands
ota-server policy print sli_zigbee_af_ota_server_policy_print
Prints the polices used by the OTA Server Policy component.
ota-server policy query u8 setPolicy
Sets the policy used by the OTA Server Policy component when it receives a query request from the client.
Arguments
  • u80 Upgrade if server has newer (default), 1 Downgrade if server has older, 2 Reinstall if server has same, 3 No next version (no next image is available for download)
ota-server policy upgrade u8 setPolicy
Sets the policy used by the OTA Server Policy component when it receives an upgrade end request.
Arguments
  • u80 Upgrade Now (default), 1 Upgrade in 2 minutes, 2 Ask me later to upgrade, 3 Abort Upgrade (send default response)
ota-server policy page-req-miss u8 setPolicy
Sets the modulus number of blocks not to respond to. This is used in testing to simulate a device that fails to receive certain blocks from an Image Page Request message.
Arguments
  • u8The block modulus number to skip sending when responding to an Image Page Request. E.g. if 2, every other block will not be sent. 0 turns the feature off.
ota-server policy page-req-sup u8 setPolicy
Sets whether the Page Request feature is supported or not.
Arguments
  • u81 if Page Request is supported, 0 if unsupported
ota-server policy block-request u8 setPolicy
Sets the policy used by the ota-server Policy component when it receives an image block request.
Arguments
  • u80 Send block (default), 1 Delay download once for 2 minutes, 2 Always abort download after first block
ota-server policy image-req-min-period u16 setPolicy
Sets the Minimum Block Period.
Arguments
  • u16The minimum block period in milliseconds
